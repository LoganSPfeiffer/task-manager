# Git Rules

- After completing any task, always: git add . && git commit -m "descriptive message" && git push
- Commit messages should be clean and professional (no class/module references)
- Good examples: "Add task filtering and status badges", "Implement localStorage persistence", "Add sidebar navigation"
- Bad examples: "Module 2: Rendering techniques", "Section 5 homework"
- Never leave uncommitted work at end of a prompt
- Push to origin main after every commit
